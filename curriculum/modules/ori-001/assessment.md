# Assessment: status, evidence, and placement

## Instructions

Complete this assessment independently. You may consult the catalog and status
page, but do not copy the worked example or lab classifications. Submit one
readable document containing all answers, the catalog snapshot date or revision
you used, and links or paths to any assessment evidence you claim.

This assessment covers:

- **ORI-001-01:** Distinguish released, planned, and blocked curriculum
  material.
- **ORI-001-02:** Classify a test, proof, assumption, and proposal without
  strengthening its claim.
- **ORI-001-03:** Produce a personal path that begins at the earliest unpassed
  assessment.

## Knowledge check

Answer in your own words.

1. A module has stable outcomes and prerequisites, but two required learner
   files are missing. Classify its curriculum status and explain whether a
   learner may count it as completed.
2. A module has complete draft prose, but it depends on an Orange capability
   that does not exist. Classify its status and explain why “planned” would be
   insufficient.
3. Explain why a released curriculum module and an implemented Orange feature
   are different claims.
4. Classify each statement as test, proof, assumption, or proposal, and state
   one limit:
   - “On revision R, command C processed input I, printed O, and exited 0.”
   - “From premises P1 and P2, the listed inference rules derive Q.”
   - “The result relies on the cryptographic hash tool being correctly
     implemented.”
   - “The next edition should reject this source form.”

## Independent task

Produce a three-part orientation record.

1. **Status audit — ORI-001-01.** Select one released, one planned, and one
   blocked module from the catalog. For each, record its ID, title, exact
   status, prerequisites, what a learner may do now, and what would be an
   overclaim.
2. **Evidence audit — ORI-001-02.** Consider this packet:

   > At revision R, <code>orangec check sample.or</code> exited 0. A written
   > derivation concludes Q from premise P. The team relies on the filesystem
   > returning the bytes that were saved. The team suggests adding evaluation
   > in a future milestone.

   Classify all four items. Write one bounded claim supported by the test,
   list the proof's premise and scope, expose the assumption, and state why the
   proposal is not current behavior. Name at least three things the packet does
   not establish.
3. **Personal path — ORI-001-03.** List released modules in prerequisite order
   from <code>ori-001</code>. Beside each attempted assessment, record pass,
   not passed, or not attempted and cite actual evidence for every pass. Begin
   your path at the earliest assessment without passing evidence. List the next
   released steps and identify the first planned or blocked boundary. If this
   assessment is still awaiting a grade, mark <code>ori-001</code> pending and
   also state how the path changes if it passes.

## Completion criteria

Use [the rubric](rubric.md). To pass, earn at least 80/100 and pass every
critical criterion. Your submission must:

- classify curriculum availability accurately for **ORI-001-01**;
- preserve the boundaries among test, proof, assumption, and proposal for
  **ORI-001-02**;
- begin the path at the earliest assessment without pass evidence and respect
  every prerequisite for **ORI-001-03**; and
- cite the catalog snapshot used so the result can be checked.

Reading the lesson or completing the guided lab is not assessment evidence.
